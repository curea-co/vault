# 볼트 작업 가이드

이 문서는 AI(Claude)가 이 볼트에서 작업할 때 **가장 먼저 읽어야 할 지침서**입니다.

---

## 볼트의 목적

이 볼트는 팀이 학습한 지식과 업무 경험을 기록하고, 서로 연결하여 하나의 지식 그래프를 형성하는 공간입니다.

개별 문서가 아닌 **문서 간의 연결**이 핵심 가치입니다. 새로운 문서를 작성할 때마다 기존 문서와의 관계를 `[[링크]]`로 명시하여, 지식이 고립되지 않고 그래프 위에서 흐르도록 합니다.

---

## 그래프 운영 원칙

### 1. 모든 문서는 연결되어야 합니다

- 고립된 문서(어디에도 링크되지 않은 문서)는 없어야 합니다.
- 새 문서를 작성하면 **반드시 1개 이상의 기존 문서와 상호 링크**합니다.
- 기존 문서에서도 새 문서로의 링크를 추가하여 양방향 연결을 유지합니다.

### 2. 링크는 관계를 설명해야 합니다

```markdown
## 관련 문서

- [[Docker]] - 쿠버네티스가 오케스트레이션하는 컨테이너 기술
- [[ECS]] - AWS 네이티브 컨테이너 오케스트레이션 서비스와의 비교
```

단순 나열이 아닌, 현재 문서와의 **관계를 한 줄로 설명**합니다.

### 3. 영역 간 링크를 권장합니다

`learning/`의 개념 문서와 `work/`의 프로젝트 문서가 서로 연결되면 "이론과 실무"가 이어집니다.

```markdown
<!-- work/projects/인증 시스템 개선.md -->
## 관련 문서
- [[SOLID]] - 이번 리팩토링에서 적용한 설계 원칙
```

### 4. 허브 문서를 의식합니다

많은 문서에서 참조되는 허브 문서가 자연스럽게 생깁니다. 허브 문서는 해당 주제의 진입점 역할을 하므로, 품질을 높게 유지합니다.

### 5. 끊어진 링크를 방치하지 않습니다

`[[미작성 문서]]`와 같이 존재하지 않는 문서로의 링크가 발견되면, 해당 문서를 작성하거나 링크를 제거합니다.

## 볼트 구조

```
vault/
├── CLAUDE.md            ← 현재 문서 (작업 지침)
├── learning/            # 학습 지식
│   ├── _meta/           # RULES, INDEX, GLOSSARY
│   ├── _templates/      # 문서 템플릿
│   ├── concepts/        # 개념/이론 문서
│   ├── tutorials/       # 실습/가이드 문서
│   └── troubleshooting/ # 문제 해결 기록
└── work/                # 업무 기록
    ├── _meta/           # RULES, INDEX, GLOSSARY
    ├── _templates/      # 문서 템플릿
    └── projects/        # 프로젝트별 기록
```

## 공통 작성 규칙

### 문체

- **"~니다" 체의 공식 문어체**를 사용합니다.
- 구어체, 반말, 이모지는 사용하지 않습니다.

### 용어

- **한글 우선 원칙**: 의미 전달에 불필요한 영어 단어의 괄호 병기는 지양합니다.
- **영문 병기 기준**: 최초 등장 시 개념 정의가 필요한 경우에만 `한글(English)` 형식으로 병기합니다.
- 이후 동일 문서 내에서는 한글만 사용합니다.

### 프론트매터

모든 문서 최상단에 포함:

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [태그1, 태그2]
status: draft | review | done
---
```

### 링크 규칙

- 내부 링크: `[[문서명]]`, `[[문서명#섹션명]]`, `[[문서명|표시할 텍스트]]`
- 모든 문서 하단에 `## 관련 문서` 섹션 필수
- 외부 참고 자료는 `## 참고 자료` 섹션에 별도 정리

---

## 작업 전 필수 확인

1. **이 문서** - 공통 규칙 확인
2. **해당 영역의 `_meta/RULES.md`** - 세부 규칙 확인
3. **해당 영역의 `_meta/INDEX.md`** - 기존 문서 확인 (중복 방지)
4. **해당 영역의 `_meta/GLOSSARY.md`** - 용어 확인
5. **해당 영역의 `_templates/`** - 템플릿 활용
