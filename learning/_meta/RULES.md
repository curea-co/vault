# 학습 문서 작성 규칙

이 문서는 `learning` 폴더 내 문서를 작성하거나 수정할 때 따라야 할 규칙을 정의합니다. 문서 작성 전 반드시 이 규칙을 참조하여 일관된 품질을 유지합니다.

---

## 1. 문서 유형별 구조

### 1.1 개념 문서 (concepts/)

특정 개념이나 주제를 정리하는 문서입니다. 기술, 방법론, 이론 등 분야를 가리지 않습니다.

1. **정의** - 한 문장 요약 후 상세 설명
2. **등장 배경 및 필요성** - 왜 이 개념이 중요한가
3. **핵심 내용** - 주요 원리, 구성 요소, 작동 방식
4. **장점 및 이점** - 이 개념을 이해하면 얻는 효과
5. **한계점 및 고려사항** - 주의할 점, 한계
6. **적용 사례** (선택) - 실무에서 활용되는 방식
7. **관련 문서** - Obsidian 내부 링크

### 1.2 정리 문서 (notes/)

학습 과정에서 얻은 인사이트, 비교 분석, 요약 등을 자유롭게 기록하는 문서입니다. 개념 문서만큼 엄격한 구조를 요구하지 않으나, 다음은 포함합니다.

1. **요약** - 핵심 내용을 간결하게 정리
2. **본문** - 학습 내용, 비교 분석, 인사이트 등
3. **관련 문서** - Obsidian 내부 링크

---

## 2. 작성 스타일

### 2.1 문체

- 모든 문서는 **"~니다" 체의 공식 문어체**로 작성합니다.
- 구어체, 반말, 이모지는 사용하지 않습니다.

### 2.2 용어 사용

- **한글 우선 원칙**: 의미 전달에 불필요한 영어 단어의 괄호 병기는 지양합니다.
- **영문 병기 기준**: 최초 등장 시 개념 정의가 필요한 경우에만 `한글(English)` 형식으로 병기합니다.
- 이후 동일 문서 내에서는 한글만 사용합니다.
- 용어 정의는 [[GLOSSARY]]를 참조하고, 새 용어는 용어집에 추가합니다.

### 2.3 분량 및 구조

- 핵심 정보가 누락되지 않도록 하되, 불필요하게 장황한 설명은 피합니다.
- 긴 설명이 필요한 경우 문단 구분 또는 글머리 기호를 활용합니다.
- 한 섹션이 지나치게 길어지면 하위 섹션으로 분리합니다.

### 2.4 명확성

- 추측이나 모호한 표현은 피합니다.
- 정의, 구분, 비교가 필요한 경우 명확히 구분하여 설명합니다.
- "~일 수 있습니다", "~인 것 같습니다" 등의 표현은 최소화합니다.

## 3. 메타데이터 (YAML Frontmatter)

모든 문서 최상단에 다음 형식의 프론트매터를 포함합니다.

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [태그1, 태그2]
status: draft | review | done
---
```

### 3.1 태그 분류 체계

**유형 태그**
- `concept` - 개념/이론 문서
- `note` - 정리/인사이트 문서
- `reference` - 참고 자료

**주제 태그**
- 주제에 따라 자유롭게 부여합니다.
- 기존 태그와 일관성을 유지하되, 필요하면 새 태그를 추가합니다.
- 예시: `design`, `architecture`, `cloud`, `ai`, `management`, `methodology`

**상태 태그**
- `draft` - 초안 작성 중
- `review` - 검토 필요
- `done` - 완료

## 4. 링크 규칙

### 4.1 내부 링크

- 다른 문서에서 정의된 개념은 **`[[문서명]]`** 형식으로 링크합니다.
- 특정 섹션 링크: `[[문서명#섹션명]]`
- 별칭 사용: `[[문서명|표시할 텍스트]]`

### 4.2 관련 문서 섹션

모든 문서 하단에 `## 관련 문서` 섹션을 추가하고, 관련 문서들을 링크합니다.

```markdown
## 관련 문서

- [[관련개념1]] - 간단한 관계 설명
- [[관련개념2]] - 간단한 관계 설명
```

### 4.3 외부 링크

외부 참고 자료는 `## 참고 자료` 섹션에 별도 정리합니다.

```markdown
## 참고 자료

- [제목](URL) - 간단한 설명
```

## 5. 품질 기준

### 5.1 필수 충족 조건

- [ ] 프론트매터가 올바른 형식으로 포함되어 있는가
- [ ] 문서 유형에 맞는 섹션 구조를 따르는가
- [ ] 해당 분야에 익숙하지 않은 사람도 이해할 수 있는 수준인가
- [ ] 관련 문서 링크가 적절히 연결되어 있는가

### 5.2 지양 사항

- 추측성 표현 ("~인 것 같습니다", "아마도")
- 과도한 영문 병기
- 근거 없는 주장
- 지나치게 짧거나 긴 설명

---

## 6. 문서 작성 워크플로우

1. **템플릿 복사**: `_templates/` 폴더에서 적절한 템플릿을 복사합니다.
2. **프론트매터 작성**: 날짜, 태그, 상태를 설정합니다.
3. **내용 작성**: 섹션 구조에 따라 내용을 채웁니다.
4. **링크 연결**: 관련 문서를 [[링크]]로 연결합니다.
5. **용어 확인**: 새 용어가 있으면 [[GLOSSARY]]에 추가합니다.
6. **INDEX 업데이트**: [[INDEX]]에 새 문서를 등록합니다.
7. **상태 변경**: 완료 시 `status: done`으로 변경합니다.
